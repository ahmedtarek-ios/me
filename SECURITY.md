# Security Policy

## Supported Versions

This site is a static React application served via GitHub Pages. The latest commit on the `main` branch is the only supported version. Please keep your deployment in sync with `main` to receive fixes or mitigation guidance.

## Reporting a Vulnerability

1. **Email:** Send an email to `ahmed.tarek.dev@outlook.com` with the subject `SECURITY REPORT â€“ <short description>`.
2. **Details to include:**
   - Steps to reproduce (or proof-of-concept).
   - Impact assessment (what data or functionality is affected).
   - Any logs, screenshots, or supporting files.
3. **Response expectations:**
   - You should receive an acknowledgement within 72 hours.
   - Follow-up questions or mitigation timelines will be coordinated by email.

Please avoid publicly disclosing the issue until it has been confirmed and resolved. If the vulnerability involves third-party services (e.g., GitHub Pages, EmailJS, Google Maps), mention that in your report so coordinated disclosure can be arranged.

## Additional Notes

- Do not run automated scanners that could disrupt the availability of the site.
- For critical issues requiring immediate attention, mark the email as **URGENT**.
